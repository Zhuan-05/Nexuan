## **QUESTION 2 (a) FOUR phases of project life cycle**

Hmm okay… from what I remember, normally got **4 main phases**, but sometimes lecturer also mention monitoring is inside execution… so I just explain properly.

First is **Initiation phase**.  
This is like the starting point. The project idea is evaluated whether it is feasible or not. For example, for a water treatment plant in Malaysia (maybe like Sungai Langat project), they will study demand of clean water, cost, and environmental impact. Also, they prepare something called project charter… I not 100% remember all content but it is to officially approve the project.

Then **Planning phase**.  
This one I remember quite clearly because lecturer always say “fail to plan = plan to fail” 😅.  
Here they develop detailed project management plan. Like define scope, schedule, cost estimation, risk planning. For water treatment plant, they will plan treatment process (like filtration, chlorination), timeline, manpower, and budget. This phase very important because all execution depends on this.

Next is **Execution phase**.  
This is where actual work happens. Construction starts, workers on site, install pipes, pumps, treatment tanks. The project team follow the plan and produce deliverables. For example, build clarifier tank, install mechanical equipment, etc.

Then **Monitoring and Controlling phase**.  
Actually this one happen together with execution, not separate.  
They check progress, compare with plan, control cost, ensure quality. If got delay or problem, they take corrective action. Also handle change control — like if client want design change, need approval first.

Finally **Closing phase**.  
This is the last stage. The project is completed, tested (like water quality testing), then handover to client. Also close contracts, release workers, and document lessons learned. Sometimes got final report also.

## **QUESTION 2 (b) Application of PMBOK knowledge areas**

Hmm this one got **10 knowledge areas**, I try explain based on water treatment plant example.

**Integration Management**  
This one is like overall coordination. Combine all parts of project. Includes project charter, management plan, and change control. Important so project not messy.

**Scope Management**  
Define what work is included. Like what facilities in the plant — sedimentation tank, filtration system, pump house. If scope not clear, later got scope creep.

**Schedule Management**  
Plan timeline. Define activities, sequence them, estimate duration. For example, foundation work must finish before installing equipment.

**Cost Management**  
Estimate and control cost. Construction project very expensive, so must budget properly. If cost overrun, big problem.

**Quality Management**  
Ensure output meets standard. For water treatment plant, water must meet safety standards (like WHO or Malaysia standard). So need strict quality control.

**Resource Management**  
Manage manpower and materials. Like assign engineers, technicians, machines. If resource not managed properly, project delay.

**Communication Management**  
Make sure information flows properly. Like between contractor, consultant, and government. If communication poor, can cause misunderstanding.

**Risk Management**  
Identify risks like bad weather, delay in material delivery, equipment failure. Then plan mitigation. This one quite important actually.

**Procurement Management**  
Purchase materials and equipment. Like pumps, filters, chemical systems. Also manage contracts with suppliers.

**Stakeholder Management**  
Identify stakeholders like government, local community, environmental bodies. Water project affects many people, so must manage expectations.

## **QUESTION 2 (c) THREE elements of triple constraint**

Okay this one quite straightforward.

Triple constraint got **Time, Cost, Scope**.

**Time**  
Project must complete within schedule. If delay, sometimes penalty also.

**Cost**  
Must stay within budget. If overspend, project might fail financially.

**Scope**  
Define what work included. If client suddenly add more work, then time and cost will increase.

👉 These three are connected.  
If one change, others also affected. For example, if scope increase → need more time and more cost. So project manager must balance properly.

## **QUESTION 3 (a)(i) TWO types of conflicts**

First is **Task conflict**.  
This is disagreement about work. Like engineers argue about design method or materials.

Second is **Relationship conflict**.  
This is more personal. Like attitude problem or misunderstanding between team members. This one more dangerous because affect teamwork.

## **QUESTION 3 (a)(ii) FIVE conflict management strategies**

Hmm I try recall… got five:

**Avoiding**  
Ignore the conflict. Useful if small issue.

**Accommodating**  
One party give in. Maintain harmony but not always fair.

**Compromising**  
Both sides give something. Fast but not best solution.

**Competing**  
One win, one lose. Useful if urgent decision needed.

**Collaborating**  
Find win-win solution. Best method but takes more time and effort.

## **QUESTION 3 (b)(ii) Advantages and disadvantages**

**Advantages**

- Clear hierarchy, easy to know reporting line
- Specialization, each department focus on expertise
- Efficient resource usage within department

**Disadvantages**

- Poor communication between departments (silo problem)
- Decision making slow because many levels
- Less focus on project, staff more loyal to department
